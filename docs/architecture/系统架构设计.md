## 3. 概述

### 3.1 项目背景

AutonoMind 是一个具备自主决策、自我进化能力的 AI Agent 架构。它能够直接使用大模型,根据问题检索知识库,自主决策是继续检索还是使用工具执行任务,并能够自主维护和更新向量知识库。

### 3.2 核心目标

- **自主性**: Agent 能够自主决策和迭代,无需人工干预
- **智能化**: 基于 LLM 的强推理能力,实现智能知识检索和任务执行
- **进化性**: 能够自主发现新知识、识别冲突、更新知识库
- **可扩展性**: 支持多种 LLM、向量数据库和工具集成
- **可观测性**: 完整的日志、追踪和监控体系

### 3.3 技术愿景

构建一个企业级、生产可用的自主 AI Agent 平台,能够处理复杂的多轮对话和任务链,支持知识库的动态管理和自我进化。

## 4. 架构设计原则

### 4.1 SOLID 原则

- **单一职责原则(SRP)**: 每个模块只负责一个核心功能
- **开闭原则(OCP)**: 对扩展开放(新工具、新知识源),对修改关闭
- **里氏替换原则(LSP)**: 不同实现可互相替换(不同 LLM、不同向量库)
- **接口隔离原则(ISP)**: 接口小而专注,职责清晰
- **依赖倒置原则(DIP)**: 依赖抽象接口而非具体实现

### 4.2 架构模式

- **分层架构**: API 层 → Agent 层 → 核心层 → 基础设施层
- **事件驱动**: 基于消息队列的异步事件处理
- **插件化**: 工具、知识源、LLM 均可插件化扩展
- **六边形架构**: 核心业务逻辑与外部依赖解耦

### 4.3 设计原则

- **可观测性**: 所有关键操作都有日志和追踪
- **容错性**: 单点故障不影响整体系统
- **可测试性**: 核心逻辑易于单元测试和集成测试
- **性能优先**: 优先考虑响应时间和吞吐量优化

### 5.2 分层架构说明

#### 5.2.1 客户端层
- **Web UI**: 基于 Vue3 的管理界面
- **CLI**: 命令行工具,用于开发调试
- **API Client**: REST/gRPC 客户端 SDK
- **Webhook**: 外部系统集成入口

#### 5.2.2 API 网关层
- **认证授权**: JWT 鉴权、OAuth2
- **限流熔断**: 令牌桶算法、熔断器
- **路由分发**: 负载均衡、灰度发布
- **日志追踪**: 分布式追踪(OpenTelemetry)

#### 5.2.3 应用服务层
- **同步 API**: FastAPI 处理实时请求
- **异步任务**: Celery 处理长时间任务

#### 5.2.4 Agent 引擎层
- **Agent 编排器**: 协调整个 Agent 执行流程
- **决策引擎**: LLM 驱动的决策和推理
- **状态管理器**: 管理对话上下文和状态

#### 5.2.5 核心引擎层
- **知识检索引擎**: 向量检索和知识匹配
- **工具执行引擎**: 工具调用和结果处理
- **知识进化引擎**: 知识更新和冲突解决

#### 5.2.6 基础设施层
- **缓存**: Redis 缓存会话和临时数据
- **数据库**: PostgreSQL 持久化业务数据
- **消息队列**: RabbitMQ 异步通信
- **对象存储**: MinIO 存储文档和文件
- **监控**: Prometheus + Grafana 监控

### 6.3 Agent 编排器实现细节

#### 职责
- 接收用户输入,初始化 Agent 执行流程
- 协调检索、决策、执行各模块
- 管理对话上下文和历史
- 处理异常和错误恢复

#### 核心实现
```python
class AgentOrchestrator(IAgentOrchestrator):
    def __init__(
        self,
        retriever: IKnowledgeRetriever,
        decision_engine: IDecisionEngine,
        tool_runner: IToolRunner,
        evolution_engine: IEvolutionEngine
    ):
        self.retriever = retriever
        self.decision_engine = decision_engine
        self.tool_runner = tool_runner
        self.evolution_engine = evolution_engine
        self.max_iterations = 5

    async def execute(self, request: AgentRequest) -> AgentResponse:
        """执行 Agent 主流程"""
        context = {
            "session_id": request.session_id,
            "user_id": request.user_id,
            "iteration": 0
        }

        # 迭代执行
        while context["iteration"] < self.max_iterations:
            context["iteration"] += 1

            # 1. 检索知识
            knowledge = await self.retrieve_knowledge(request.query)

            # 2. 决策
            decision = await self.make_decision(request.query, knowledge)

            # 3. 执行动作
            if decision.decision_type == "complete":
                break
            elif decision.decision_type == "execute":
                result = await self.execute_action(decision)
                # 更新查询上下文
                request.query += f"\n\n执行结果: {result}"

        # 4. 生成最终响应
        response = await self._generate_response(request, knowledge)

        # 5. 知识进化(异步)
        await self._trigger_evolution_async(request, response)

        return response

    async def _trigger_evolution_async(
        self,
        request: AgentRequest,
        response: AgentResponse
    ):
        """异步触发知识进化"""
        try:
            new_knowledge = await self.evolution_engine.discover_new_knowledge({
                "query": request.query,
                "response": response.dict()
            })
            if new_knowledge:
                logger.info(f"发现 {len(new_knowledge)} 条新知识")
        except Exception as e:
            logger.error(f"知识进化失败: {e}")
```

### 6.2 决策引擎(Decision Engine)

- 基于 LLM 进行推理和决策
- 判断是继续检索还是执行工具

#### 决策策略

### 8.2 数据库分库分表策略

#### 8.2.1 分库策略

**垂直分库**:
- `user_db`: 用户表、系统配置表
- `session_db`: 会话表、消息表
- `knowledge_db`: 知识表、文档表

**水平分库**:
- 按 `user_id` 的 hash 值分库: `hash(user_id) % 4` → 4个库
- 每个库独立部署,提升并发能力

#### 8.2.2 分表策略

**会话表分表**:
```sql

    CHECK (hash(user_id)::int % 4 = 0)
) INHERITS (sessions);

CREATE TABLE sessions_1 (
    CHECK (hash(user_id)::int % 4 = 1)
) INHERITS (sessions);

CREATE TABLE sessions_2 (
    CHECK (hash(user_id)::int % 4 = 2)
) INHERITS (sessions);

CREATE TABLE sessions_3 (
    CHECK (hash(user_id)::int % 4 = 3)
) INHERITS (sessions);

-- 插入触发器自动路由
CREATE OR REPLACE FUNCTION sessions_insert_trigger()
RETURNS TRIGGER AS $$
BEGIN
    IF hash(NEW.user_id)::int % 4 = 0 THEN
        INSERT INTO sessions_0 VALUES (NEW.*);
    ELSIF hash(NEW.user_id)::int % 4 = 1 THEN
        INSERT INTO sessions_1 VALUES (NEW.*);
    ELSIF hash(NEW.user_id)::int % 4 = 2 THEN
        INSERT INTO sessions_2 VALUES (NEW.*);
    ELSE
        INSERT INTO sessions_3 VALUES (NEW.*);
    END IF;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;
```

### 8.3 数据一致性保障

#### 8.3.1 分布式事务

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

class DistributedTransaction:
    """分布式事务管理"""

    async def execute_transaction(self, operations: List[callable]):
        """执行分布式事务"""
        sessions = []
        try:
            # 开启所有数据库会话
            for db_url in self.db_urls:
                engine = create_engine(db_url)
                session = sessionmaker(bind=engine)()
                sessions.append(session)

            # 执行操作
            for op in operations:
                await op(sessions)

            # 提交所有事务
            for session in sessions:
                session.commit()

        except Exception as e:
            # 回滚所有事务
            for session in sessions:
                session.rollback()
            raise e
        finally:
            for session in sessions:
                session.close()
```

```python
        self,
```

#### 8.5.1 分层错误处理

```python
class ErrorCode(Enum):
    """错误码定义"""
    # 通用错误 1000-1999
    INTERNAL_ERROR = 1001
    TIMEOUT_ERROR = 1002

    # 认证错误 2000-2999
    UNAUTHORIZED = 2001
    TOKEN_EXPIRED = 2002

    # 参数错误 3000-3999
    INVALID_PARAMETER = 3001
    MISSING_PARAMETER = 3002

    # 业务错误 4000-4999
    SESSION_NOT_FOUND = 4001
    KNOWLEDGE_NOT_FOUND = 4002
    TOOL_EXECUTION_FAILED = 4003

    # 外部服务错误 5000-5999
    LLM_API_ERROR = 5001
    VECTOR_DB_ERROR = 5002
    QUEUE_ERROR = 5003


class AutonoMindException(Exception):
    """基础异常类"""

    def __init__(
        self,
        code: ErrorCode,
        message: str,
        details: dict = None
    ):
        self.code = code
        self.message = message
        self.details = details or {}
        super().__init__(message)


class LLMException(AutonoMindException):
    """LLM 调用异常"""

    def __init__(
        self,
        message: str,
        original_error: Exception = None
    ):
        details = {"original_error": str(original_error)} if original_error else None
        super().__init__(
            code=ErrorCode.LLM_API_ERROR,
            message=message,
            details=details
        )


class RetrieverException(AutonoMindException):
    """检索异常"""

    def __init__(
        self,
        message: str,
        query: str = None
    ):
        details = {"query": query} if query else None
        super().__init__(
            code=ErrorCode.VECTOR_DB_ERROR,
            message=message,
            details=details
        )


class ToolExecutionException(AutonoMindException):
    """工具执行异常"""

    def __init__(
        self,
        tool_name: str,
        message: str,
        parameters: dict = None
    ):
        details = {
            "tool_name": tool_name,
            "parameters": parameters
        } if parameters else {"tool_name": tool_name}
        super().__init__(
            code=ErrorCode.TOOL_EXECUTION_FAILED,
            message=message,
            details=details
        )
```

#### 8.5.2 全局异常处理中间件

```python
from fastapi import Request, status
from fastapi.responses import JSONResponse

async def exception_handler(request: Request, exc: AutonoMindException):
    """全局异常处理"""
    return JSONResponse(
        status_code=status.HTTP_400_BAD_REQUEST,
        content={
            "success": False,
            "error": {
                "code": exc.code.name,
                "message": exc.message,
                "details": exc.details
            },
            "meta": {
                "timestamp": datetime.utcnow().isoformat(),
                "request_id": request.state.request_id
            }
        }
    )


async def llm_exception_handler(request: Request, exc: LLMException):
    """LLM 异常特殊处理"""
    # LLM 调用失败时,尝试降级为规则决策
    if hasattr(request.state, "fallback_decision"):
        return JSONResponse(
            status_code=status.HTTP_200_OK,
            content={
                "success": True,
                "data": request.state.fallback_decision,
                "meta": {
                    "warning": "LLM 不可用,使用降级策略"
                }
            }
        )

    return JSONResponse(
        status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
        content={
            "success": False,
            "error": {
                "code": "LLM_SERVICE_UNAVAILABLE",
                "message": "LLM 服务暂时不可用"
            }
        }
    )


# 注册异常处理器
```

#### 8.5.3 重试机制

```python
from tenacity import retry, stop_after_attempt, wait_exponential

class LLMService:
    """LLM 服务,带重试机制"""

    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),
        retry=retry_if_exception_type(ConnectionError)
    )
    async def call_llm(self, prompt: str) -> str:
        """调用 LLM,自动重试"""
        try:
            response = await self.client.chat.completions.create(
                model=self.model,
                messages=[{"role": "user", "content": prompt}]
            )
            return response.choices[0].message.content
        except ConnectionError as e:
            logger.warning(f"LLM 连接失败,重试中... {e}")
            raise
        except Exception as e:
            logger.error(f"LLM 调用失败: {e}")
            raise LLMException("LLM 调用失败", e)
```
